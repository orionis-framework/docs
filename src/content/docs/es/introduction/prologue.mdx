# Orionis Framework

## El framework async-first que Python nunca tuvo.

*One framework. Zero compromises.*

![Orionis Framework Logo](/prologue/logo.png)

:::tip[Async-first. Potenciado por Rust. Hecho para producción.]
Orionis es un framework completo para construir **APIs, aplicaciones web, sockets, colas, jobs, tareas programadas y comandos CLI** — con la arquitectura y experiencia de desarrollo que Python necesitaba.
:::

:::note[No es otro framework más]
Orionis no es un router, ni un microframework, ni un toolkit. Es un **framework full-stack async-first**, diseñado para reemplazar la necesidad de ensamblar múltiples herramientas en Python.
:::

---

## El Problema No Es Python. Son Las Herramientas.

La mayoría de frameworks Python te dan un router y te dejan el resto a ti. Terminas ensamblando decenas de librerías, escribiendo boilerplate y peleando con decisiones que deberían estar resueltas.

**¿El resultado?** Estructuras de proyecto inconsistentes. Sin convenciones claras. Gestión de dependencias como un afterthought. Testing que se siente como una carga.

Orionis toma un camino diferente.

:::note[Un framework, no un toolkit]
Orionis incluye todo lo que necesitas — inyección de dependencias, service providers, middleware, facades, un CLI y un motor de testing — diseñados para funcionar juntos desde el primer día. Un framework. Cero código de pegamento.
:::

## Por Qué Orionis

<CardGrid>
  <Card title="Núcleo HTTP Potenciado por Rust" icon="rocket">
    Construido sobre **Granian**, el servidor HTTP más rápido del ecosistema Python. Soporte ASGI y RSGI de serie. **10x más rápido** que los servidores Python tradicionales.
  </Card>

  <Card title="Arquitectura Async-First" icon="star">
    Diseñado desde cero para `async/await`. No añadido después. No opcional. Cada capa — desde routing hasta middleware y DI — es nativamente asíncrona.
  </Card>

  <Card title="Reactor CLI" icon="setting">
    Una interfaz de línea de comandos potente para scaffolding, programación de tareas y procesamiento de jobs. Ejecuta `python -B reactor schedule:work` y despliega.
  </Card>

  <Card title="Motor de Testing Integrado" icon="document">
    Testing de primera clase con assertions expresivos, soporte async y ejecución en paralelo. Sin necesidad de test runners externos.
  </Card>

  <Card title="Arquitectura Limpia que Escala" icon="open-book">
    Service providers, pipelines de middleware, inyección de dependencias y facades. Patrones que mantienen tu código limpio con 100 rutas o con 10,000.
  </Card>

  <Card title="Seguro por Defecto" icon="approve-check">
    Seguridad alineada con OWASP, protección por middleware integrada y primitivas de autenticación. La seguridad no es un plugin — es la base.
  </Card>
</CardGrid>

## Rendimiento que Habla por Sí Solo

:::caution[Medido. No Estimado.]
Orionis corre sobre **Granian** — un servidor HTTP escrito en Rust. Esto no es rendimiento teórico. Son números reales en hardware real.
:::

### Elige tu Interfaz

<Tabs>
  <TabItem label="ASGI" icon="laptop">
    **ASGI** *(Asynchronous Server Gateway Interface)*

    Compatibilidad total con ASGI significa que Orionis se integra sin fricción con el ecosistema async de Python. Usa las herramientas que ya conoces — con el rendimiento que siempre quisiste.
  </TabItem>

  <TabItem label="RSGI" icon="rocket">
    **RSGI** *(Rust Server Gateway Interface)*

    Ve más allá de ASGI. RSGI es la interfaz nativa de Rust en Granian — desbloqueando el máximo throughput y la menor latencia posible. Para cuando cada microsegundo importa.
  </TabItem>
</Tabs>
